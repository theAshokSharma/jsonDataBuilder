üéØ Overview

The options file is a JSON configuration file that defines custom input controls, validation rules, and behavior for your form fields. It works alongside your JSON schema to provide a rich, interactive form-building experience.

üí° Tip: The options file is optional but highly recommended for better user experience and data validation.
üí° Key Benefits:
  • Define custom dropdown options for form fields
  • Control single vs. multi-select behavior
  • Define input controls such as sliders, date-time pickers, etc.
  • Implement dynamic field dependencies
  • Create conditional enable/disable logic
  • Add exclusive selection options

üìê File Structure

An options file consists of field configurations mapped to schema paths:

{
  "field.path.name": {
    "values": ["default1", "default2"],
    "input_control": "drop-down",
    "response_type": "single-select",
    "dependent_values": {
      "trigger_field": {
        "value1": ["opt1", "opt2"],
        "value2": {                    
          "values": ["opt3", "opt4"],
          "input_control": "radio-button",
          "response_type": "single-select"
        }
      }
    }
  },
  "conditional_rules": {
      "field.path.name": [
          {
              "value": "some value",
              "disable_fields": [
                  "field1.path.name",
                  "field2.path.name",
                  "field3.path.name"
              ]
          },
          {
              "value": "some other value",
              "disable_fields": [
                  "field5.path.name",
                  "field6.path.name"
              ]
          }
        ]
  }
}

Required Properties

Property Type Description
input_control string Type of input control: "drop-down", "check-box", "radio-button", "slider", "date-time-picker"
values array List of available options (strings or objects with value/label)

Optional Properties

Property Type Applies To Description
response_type string drop-down only "single-select" or "multi-select"
na string all Value for "Not Applicable" option
exclusive_values array multi-select Values that deselect all others when chosen
dependent_values object all Dynamic options based on another field

üéõÔ∏è Input Controls

The JSON Data Builder supports five types of input controls, each optimized for different use cases.

üìã Drop-down Lists

Drop-down lists are ideal for selecting from a predefined list of options.

Single-Select Drop-down

{
  "demographic.country": {
    "input_control": "drop-down",
    "response_type": "single-select",
    "values": ["USA", "Canada", "Mexico", "UK"],
    "na": "Prefer not to say"
  }
}

Multi-Select Drop-down

{
  "medical_history.conditions": {
    "input_control": "drop-down",
    "response_type": "multi-select",
    "values": [
      "Diabetes",
      "Hypertension",
      "Heart Disease",
      "Asthma",
      "None of the above"
    ],
    "exclusive_values": ["None of the above", "N/A"],
    "na": "N/A"
  }
}
Note: The response_type property only applies to drop-down controls. For other input types, it is ignored.

Check boxes allow multiple selections and are displayed as a list of checkable items.

Basic Check Box

{
  "preferences.notifications": {
    "input_control": "check-box",
    "values": [
      "Email notifications",
      "SMS notifications",
      "Push notifications",
      "Newsletter"
    ]
  }
}

Check Box with Labels

{
  "lifestyle.activities": {
    "input_control": "check-box",
    "values": [
      { "value": "running", "label": "Running/Jogging" },
      { "value": "swimming", "label": "Swimming" },
      { "value": "cycling", "label": "Cycling/Biking" },
      { "value": "gym", "label": "Gym Workouts" },
      { "value": "yoga", "label": "Yoga/Pilates" }
    ],
    "na": "None"
  }
}

Check Box with Exclusive Values

{
  "medical_history.allergies": {
    "input_control": "check-box",
    "values": [
      "Peanuts",
      "Shellfish",
      "Dairy",
      "Eggs",
      "Soy",
      "No known allergies"
    ],
    "exclusive_values": ["No known allergies"]
  }
}
üí° Tip: Use check boxes when users need to see all options at once and may select multiple items.

üéöÔ∏è Sliders

Sliders provide a visual way to select a value from a continuous or discrete range.

Numeric Range Slider

{
  "lifestyle.exercise_hours": {
    "input_control": "slider",
    "values": ["0-20"],
    "min": 0,
    "max": 20,
    "step": 1,
    "unit": "hours per week"
  }
}

Discrete Value Slider

{
  "assessment.pain_level": {
    "input_control": "slider",
    "values": [
      { "value": "0", "label": "No Pain" },
      { "value": "2", "label": "Mild" },
      { "value": "5", "label": "Moderate" },
      { "value": "8", "label": "Severe" },
      { "value": "10", "label": "Worst Possible" }
    ],
    "min": 0,
    "max": 10,
    "step": 1
  }
}

Slider Properties

Property Type Description
min number Minimum value (default: 0)
max number Maximum value (default: 100)
step number Increment value (default: 1)
unit string Display unit (e.g., "hours", "kg", "%")

üè∑Ô∏è Values Configuration

The values property supports multiple formats for maximum flexibility.

Simple String Values

Use simple strings when the display text and stored value should be the same:

{
  "colors": {
    "input_control": "check-box",
    "values": ["Red", "Green", "Blue", "Yellow"]
  }
}

Value/Label Object Pairs

Use objects with value and label properties when you need different stored values and display text:

{
  "country": {
    "input_control": "drop-down",
    "response_type": "single-select",
    "values": [
      { "value": "US", "label": "United States üá∫üá∏" },
      { "value": "CA", "label": "Canada üá®üá¶" },
      { "value": "MX", "label": "Mexico üá≤üáΩ" },
      { "value": "UK", "label": "United Kingdom üá¨üáß" }
    ]
  }
}

Range Notation

Use range notation for numeric sequences:

{
  "age": {
    "input_control": "drop-down",
    "response_type": "single-select",
    "values": ["18-100"]
  }
}

This expands to: ["18", "19", "20", ... "99", "100"]

Mixed Formats

You can mix different value formats in a single array:

{
  "education": {
    "input_control": "radio-button",
    "values": [
      { "value": "hs", "label": "High School Diploma" },
      { "value": "associate", "label": "Associate Degree" },
      { "value": "bachelor", "label": "Bachelor's Degree" },
      { "value": "master", "label": "Master's Degree" },
      { "value": "doctorate", "label": "Doctorate/PhD" },
      "Other"
    ]
  }
}
‚ö†Ô∏è Important: When using value/label objects, the label property must immediately follow the value property in the object definition.

üöÄ Advanced Features

Exclusive Values

Exclusive values automatically deselect all other options when selected. Useful for "None" or "N/A" options:

{
  "symptoms": {
    "input_control": "check-box",
    "values": [
      "Fever",
      "Cough",
      "Headache",
      "Fatigue",
      "None of the above"
    ],
    "exclusive_values": ["None of the above"],
    "na": "Prefer not to say"
  }
}

Dependent Values

Dependent values dynamically change options based on another field's value:

{
  "demographic.gender": {
    "input_control": "radio-button",
    "values": ["Male", "Female"]
  },
  "medical_history.cancer_types": {
    "input_control": "check-box",
    "values": [
      "Breast Cancer",
      "Prostate Cancer",
      "Lung Cancer",
      "Colorectal Cancer"
    ],
    "dependent_values": {
      "demographic.gender": {
        "Male": [
          "Prostate Cancer",
          "Lung Cancer",
          "Colorectal Cancer"
        ],
        "Female": [
          "Breast Cancer",
          "Lung Cancer",
          "Colorectal Cancer",
          "Ovarian Cancer"
        ]
      }
    }
  }
}

N/A Option

The na property adds a special "Not Applicable" option:

{
  "employment.industry": {
    "input_control": "drop-down",
    "response_type": "single-select",
    "values": [
      "Healthcare",
      "Technology",
      "Education",
      "Finance",
      "Manufacturing"
    ],
    "na": "Unemployed"
  }
}

üîÄ Conditional Rules

Conditional rules enable fields to be automatically disabled based on other field values:

Basic Conditional Rule

{
  "lifestyle.smoke_cigarette": {
    "input_control": "radio-button",
    "values": ["Yes", "No", "I Quit"]
  },
  "lifestyle.cigarettes_per_day": {
    "input_control": "slider",
    "values": ["1-50"],
    "min": 1,
    "max": 50,
    "step": 1
  },
  "conditional_rules": {
    "lifestyle.smoke_cigarette": [
      {
        "value": "No",
        "disable_fields": [
          "lifestyle.cigarettes_per_day",
          "lifestyle.years_smoking"
        ]
      }
    ]
  }
}

Multiple Conditional Rules

{
  "demographic.gender": {
    "input_control": "radio-button",
    "values": ["Male", "Female"]
  },
  "conditional_rules": {
    "demographic.gender": [
      {
        "value": "Male",
        "disable_fields": [
          "medical_history.pregnancy_history",
          "medical_history.menstrual_cycle",
          "medical_history.menopause_status"
        ]
      },
      {
        "value": "Female",
        "disable_fields": [
          "medical_history.prostate_exam",
          "medical_history.testicular_exam"
        ]
      }
    ]
  }
}

Conditional Rules Structure

Property Type Description
value string The trigger value that activates the rule
disable_fields array List of field paths to disable when condition is met
üí° Tip: Disabled fields are automatically set to default values (N/A, -9999, or 1900-01-01 depending on type).

‚úÖ Best Practices

1. Choose the Right Input Control

  • Drop-down: 8+ options, limited space
  • Radio buttons: 2-7 options, mutually exclusive
  • Check boxes: Multiple selections needed, <10 options
  • Slider: Numeric ranges or ordered scales
  • Date-time picker: Any date/time input

2. Use Value/Label Pairs When Needed

Use objects with value and label when:

  • Stored value differs from display text (e.g., "US" vs "United States")
  • Adding icons or emojis to labels
  • Creating more descriptive display text

3. Leverage Conditional Rules

Use conditional rules to:

  • Simplify complex forms
  • Prevent invalid data combinations
  • Improve user experience by hiding irrelevant fields

4. Use Dependent Values for Dynamic Forms

Dependent values are perfect for:

  • Gender-specific medical fields
  • Country-specific state/province lists
  • Category-specific subcategories

5. Validate Your Options File

Always validate your options file against your schema:

  • Use the built-in validation when loading files
  • Ensure all field paths match schema exactly
  • Test conditional rules thoroughly

üîç Troubleshooting

Common Issues

Issue: Field not showing custom options
Solution: Verify field path exactly matches schema path (case-sensitive)
Issue: Conditional rules not working
Solution: Check that trigger field values match exactly (including whitespace)
Issue: Dependent values not updating
Solution: Ensure dependent field name matches trigger field path exactly
Issue: Slider not displaying correctly
Solution: Verify min, max, and step values are numeric and logical
Issue: Date picker not restricting dates
Solution: Check min/max format matches the picker format type
